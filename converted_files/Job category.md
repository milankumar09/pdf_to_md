How to Update Proto Files to Latest Schema(Job) Version :

Create new proto files for latest Job schema

- Step1 : Copy all the dependent json schema files for the main job.version schema into a folder as convenient as shown below (schema files source : https://schemaregistry.cso-hp.com/browse)
- Step2 : Create a folder to save the newly created .proto files say ~/protofiles
- Step3 : Create a folder "ipc" in the path below to copy the new protofiles required for Trillium ..dune/src/fw/ares/framework/JobTracker/Standard/ipc
- Step4 : cd to /sirius/tools/cdmProtoGen - this is where the script cdmProtoGen, which creates the proto files - is present along with a README file to understand the command to use the script cdmProtoGen
- Step5 : Run the below Command to create proto file for the main schema i.e., job.version :
  - ./cdmProtoGen -s ~/JobSchema -i ~/JobSchema/com.hp.cdm.domain.telemetry.type.eventDetail.category.job.version.1.schema.json -f ~/protofiles
  - Creates the file in ~/protofiles telemetry.jobCategoryJobVersion1_eventDetail.proto
- Step6 : Now this file won't have all the details as the schema jason for job.version will interdepend on jobManagement schema json for more details of printinfo, scan, job and client info as seen in the job.version schema. So create another proto file for jobManagement with below command :
  - ./cdmProtoGen -s ~/JobSchema -i ~/JobSchema/com.hp.cdm.service.jobManagement.version.1.sharedTypes.jobManagement.schema.json -f ~/protofiles
  - Creates the file in ~/protofiles jobManagementSharedTypesJobManagement_1.proto
- Step7 : Copy the newly generated proto files and place it under ..dune/src/fw/ares/framework/JobTracker/Standard/ipc
- Step8 : Build Trillium firmware ./run make_target.py trillium_lp1 and start solving the build errors
  - This stage might ask for the proto files which are imported inside the new proto files created above
  - Make sure to create those inter-dependent proto files for respective latest versions and check-in all the proto files generated

Note : One example proto update to schema version 1.5 for Trillium is mentioned in detail under for reference. - DUNE-208376 Getting issue details... STATUS
